# FAQ: Technical

## Is there a mobile app?

Currently, Dayopt is a web application optimized for both desktop and mobile browsers. As a PWA (Progressive Web App), you can add it to your home screen for an app-like experience. A native mobile app is on the roadmap.

## Which browsers are supported?

All major modern browsers — Chrome, Firefox, Safari, and Edge. We support the latest two versions of each.

## Does it ever get slow?

We pay close attention to performance. We maintain Core Web Vitals scores above 90 and prioritize a snappy experience. If you ever notice slowness, please share your feedback.

## What can I do with the API?

Create, retrieve, and update timeboxes programmatically. Build your own automation workflows or integrate with other tools in your stack.

## Do you support Webhooks?

Please check our documentation for current API capabilities and planned extensions. If you have a specific need, your feedback helps us prioritize development.

## Can I import data from other tools?

For data migration from other tools, please reach out. We'll support as smooth a transition as possible.

## Where can I check for outages or maintenance?

We communicate service status transparently through our status page and announcements. Planned maintenance is announced in advance.

## How is accessibility handled?

We develop toward WCAG 2.1 AA compliance — keyboard navigation, screen reader support, and sufficient contrast ratios. We aim to be usable for everyone.

## Is Dayopt available in multiple languages?

Currently English and Japanese. All UI text is available in both languages. Additional language support will be considered based on user feedback.

## Can I self-host Dayopt?

Currently available as a cloud service only. Self-hosting may be considered based on demand.
